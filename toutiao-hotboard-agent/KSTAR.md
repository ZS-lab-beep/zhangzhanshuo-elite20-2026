# K-S-T-A-R 记录：今日头条热榜自动抓取 Agent

## K — Knowledge（知识）

### 领域知识
- **今日头条热榜数据接口**: `https://www.toutiao.com/hot-event/hot-board/?origin=toutiao_pc`
  - 返回 JSON 格式数据，包含 Title / HotValue / Label / Image / InterestCategory 等字段
  - HotValue 为热度数值，可用于排序
  - Label 标识特殊状态：`new`（新上榜）、`hot`（热门）、`refuteRumors`（辟谣）
  - ClusterType 标识内容类型（0=图文, 2=话题, 4=财经, 6=社会等）
- **头条 PC 端热榜页**: `https://www.toutiao.com/trending/` 已失效（404）
- **头条内容安全性**: 页面前端需要登录 / SSR 渲染，但数据 API 无需认证即可调用

### 通用知识
- **Web Fetch 工具能力边界**: 可抓取 HTTP 响应内容，支持 JSON 解析模式；不支持 JavaScript 渲染
- **JSON 数据解析**: 需要从嵌套结构中提取关键字段，忽略元数据（URL/Image/LabelUri 等）
- **API 截断处理**: 单次获取有限制（~50KB / ~17条），超出部分需分批次或多 URL 获取
- **中文自然语言呈现**: 热榜输出需要结构化、带序号、过滤低价值字段

## S — Skill（技能）

### 技术技能
- ✅ **网页数据抓取**: 使用 web_fetch 工具获取 API JSON 数据
- ✅ **URL 探测与试错**: 主 URL 404 时自动检测并切换备选接口
- ✅ **JSON 解析与信息提取**: 从深层嵌套 JSON 中提取 Title、HotValue、Label 核心字段
- ✅ **数据排序与聚合**: 按 HotValue 降序排列
- ✅ **输出格式设计**: 序号 + 标题 + 简要说明 + 特殊标签（🔥 ⚡新 🏷辟谣）

### 软技能
- **意图理解**: 从"帮我看看热榜"这种口语化指令中识别出具体的抓取 + 总结需求
- **结果呈现**: 简洁、分类清晰、不带冗余技术信息
- **容错处理**: 页面错误时自动尝试其他路径

## A — Approach（方法）

### Step-by-step 处理流程

```
[1] 接收用户请求
    ↓ 识别意图 → "今日头条/热榜/今天的新闻"
    
[2] URL 选择
    ↓ 尝试优先路径: https://www.toutiao.com/hot-event/hot-board/
    ↓ 失败回退: 无登录需求的 JSON 接口
    
[3] 请求参数
    ↓ extractMode = "text" (获取原始 JSON)
    ↓ maxChars 根据数据量动态调整（首次 15K → 不足时扩大到 50K）
    
[4] 数据解析
    ↓ 从 data[] 数组中遍历每条记录
    ↓ 提取: Title（标题）、HotValue（热度值）、Label（标签）、ClusterType（类型）
    ↓ 忽略: Image / LabelUri / Schema / 追踪参数log_pb
    
[5] 结果生成
    ↓ 按 HotValue 降序排列
    ↓ 标记特殊标签（⚡新 = new, 🔥 = hot, 🏷辟谣 = refuteRumors）
    ↓ 生成序号列表 + 简要解读
    
[6] 呈现给用户
    ↓ 自然语言输出，分节、分类
```

### 决策树

```
请求到达
├─ 知道 URL → 直接用
├─ 不知道 URL 但有站点名 → 凭站点知识推断
│   ├─ 成功 → 继续
│   └─ 404 → 尝试已知备选路径
│       ├─ 成功 → 继续
│       └─ 失败 → 告知用户并提供替代方案
└─ 完全不知道 → 告知无法自动发现
```

- ✅ **结构化中文总结**：带序号、热度排序、特殊标签
- ✅ **零人工干预**：无需用户提供 URL 或手动操作

- ⚠️ 依赖于 API 接口结构不变，如改版则需更新
